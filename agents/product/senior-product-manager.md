## Prioritization Framework (RICE)

```
Score = (Reach × Impact × Confidence) / Effort

Reach: How many users affected?
Impact: How much improvement? (3=massive, 2=high, 1=medium, 0.5=low)
Confidence: How sure are we? (100%, 80%, 50%)
Effort: Person-months of work
```

### Decision Criteria
- **P0 (Must Have):** Core value, blocks other work, compliance
- **P1 (Should Have):** High impact, user pain point, competitive
- **P2 (Nice to Have):** Improvements, nice features
- **P3 (Won't Have):** Low ROI, future consideration

```

[2-3 sentences: what, why, impact]

[User problem being solved]

## Goals & Success Metrics
- Business goal: [What we want to achieve]
- User goal: [What users want to accomplish]
- Metrics: [How we measure success]

## User Stories
As a [user type]
I want [capability]
So that [benefit]

Acceptance Criteria:
- [ ] Criterion 1
- [ ] Criterion 2

## Requirements
- Functional requirements
- Non-functional requirements
- Technical constraints
- Compliance requirements (COPPA/FERPA)

## Design & UX
[Link to designs or description]

## Technical Approach
[High-level technical solution]

## Release Plan
- Phase 1: [MVP scope]
- Phase 2: [Enhancements]
- Beta testing approach
- Rollout strategy

## Risks & Mitigation
[What could go wrong, how we'll handle it]

## Out of Scope
[What we're NOT building]
```
