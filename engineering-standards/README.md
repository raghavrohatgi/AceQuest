## 📋 Key Principles

### 1. **Type Safety First**
- TypeScript strict mode always
- No `any` types
- Validate all external inputs

### 2. **Security by Default**
- Validate all inputs
- Sanitize user content
- Use parameterized queries
- HTTPS only in production
- Proper authentication & authorization

### 3. **Accessibility Required**
- WCAG 2.1 AA compliance
- Semantic HTML
- Keyboard navigation
- Screen reader support
- Age-appropriate for grades 3-8

### 4. **Performance Matters**
- Code splitting
- Image optimization
- Caching strategy
- Database query optimization
- API response time <200ms

### 5. **Test Everything**
- Unit tests for business logic
- Integration tests for APIs
- E2E tests for critical flows
- 80%+ code coverage

### 6. **Document Clearly**
- Complex components documented
- API endpoints documented
- Architecture decisions recorded
- README files everywhere

## 📖 Quick References

### Frontend Quick Tips

```typescript
// ✅ Component structure
export function ComponentName({ prop }: Props) {
  // 1. Hooks at top
  // 2. Derived state
  // 3. Event handlers
  // 4. Effects
  // 5. Return JSX
}

// ✅ Always validate props
interface Props {
  id: string;
  required: number;
  optional?: boolean;
}

// ✅ Use semantic HTML
<button type="button" aria-label="Description">
  Click Me
</button>
```

### Backend Quick Tips

```typescript
// ✅ Controller pattern
async function handler(req, res, next) {
  try {
    // 1. Validate input
    const data = schema.parse(req.body);

    // 2. Call service
    const result = await service.doWork(data);

    // 3. Return response
    return res.json({ success: true, data: result });
  } catch (error) {
    next(error); // Let error handler deal with it
  }
}

// ✅ Always use transactions for related operations
await prisma.$transaction(async (tx) => {
  const user = await tx.user.create({...});
  await tx.profile.create({...});
});
```

### API Quick Tips

```typescript
// ✅ RESTful routes
GET    /resource          # List
POST   /resource          # Create
GET    /resource/:id      # Get one
PATCH  /resource/:id      # Update
DELETE /resource/:id      # Delete

// ✅ Consistent responses
{
  success: boolean,
  data?: T,
  error?: { code, message }
}
```
