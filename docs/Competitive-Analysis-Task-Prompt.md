# Competitive Analysis Task Prompt

## Task Overview
Create comprehensive, individual competitor analysis files for each major competitor in the assessment/EdTech space. Each competitor gets their own dedicated markdown file with in-depth research, analysis, and strategic insights.

---

## Task Requirements

### 1. Research Phase
For each competitor, conduct web research to gather:
- Official website URL and company information
- Current products and services offered
- Pricing structure (both B2C and B2B if applicable)
- User reviews and ratings from multiple sources (Trustpilot, G2, MouthShut, etc.)
- School/client implementations (if available)
- Recent news, funding rounds, or significant changes
- Y Combinator or accelerator background (if applicable)

### 2. File Structure
Create one markdown file per competitor with the following sections:

#### Required Sections:

**A. Company Overview**
- Company name, founded date, founders
- Headquarters location
- Official website URL
- Type/category of platform
- Market position and description

**B. Market Segment**
- Primary target audience (grades/age groups)
- Business model (B2C, B2B, B2B2C)
- Geographic focus
- Customer type breakdown
- Target audience demographics

**C. Products and Services**
- Detailed list of all products offered
- Key features of each product
- Curriculum alignment (CBSE, ICSE, State Boards)
- Platform availability (web, mobile, tablet)

**D. Pricing Structure**
- Current pricing (create tables for clarity)
- B2C pricing tiers
- B2B/school pricing (if available)
- Payment options (EMI, installments, discounts)
- Free tier or trial information
- Historical pricing changes (if significant)

**E. Key Features**
- Assessment capabilities
- Learning features (if applicable)
- Technology platform details
- Unique selling propositions

**F. Strengths**
- 5-7 major strengths
- Each with explanation and impact
- Numbered list format
- Evidence-based (cite reviews, metrics)

**G. Weaknesses**
- 7-10 major weaknesses
- Each with detailed explanation
- Include user complaints and evidence
- Quote actual user feedback where possible
- Numbered list format

**H. User Sentiment and Reviews**
- Overall ratings table from multiple platforms
- Common positive feedback (with ✅ bullets)
- Common negative feedback (with ❌ bullets)
- Specific user complaints with quotes
- Net sentiment summary

**I. School Clients and Reach** (if applicable)
- Documented school partnerships
- Number of schools using platform
- Geographic reach
- Notable implementations
- Impact metrics (students reached)

**J. AceQuest Differentiation Opportunities**
- 7-10 specific differentiation strategies
- Format: "Competitor Problem" → "AceQuest Advantage"
- Each opportunity with clear explanation
- Numbered list format
- Strategic and actionable

**K. Competitive Threat Level**
- Overall threat rating: 🔴 High / 🟡 Moderate / 🟢 Low
- "Why [Threat Level]" explanation
- Potential threats bullet list
- Mitigation strategy
- Clear reasoning for assessment

**L. Strategic Insights**
- Market opportunity analysis
- Positioning recommendations
- Customer acquisition strategies
- Target audience profiles
- Key takeaways for AceQuest strategy

**M. Sources and References**
- All website URLs cited
- Review platform links
- News articles and reports
- Date accessed

**N. Footer**
- Last updated date
- Analyst credit line

### 3. Additional Files Required

**Index/README File**
Create a comprehensive README.md that includes:
- Overview of all competitors
- Summary comparison table
- Quick reference for each competitor
- Market gaps and opportunities
- Competitive positioning matrix
- Strategic recommendations
- Target customer profiles
- Acquisition channel recommendations
- Monitoring and update schedule

## Differentiation Framework

For each competitor weakness, identify:

1. **The Problem**: What specific issue does the competitor have?
2. **Customer Impact**: How does this affect users?
3. **AceQuest Advantage**: How can AceQuest solve this better?
4. **Implementation**: What feature/approach will deliver this advantage?
5. **Value Proposition**: How to message this to customers?

### Example Format:
```markdown
### 1. [Weakness Category]
**Competitor Problem:** [Specific issue with evidence]
**AceQuest Advantage:**
- [Solution point 1]
- [Solution point 2]
- [Solution point 3]
- [Value proposition]
- [Competitive moat]
```

## Threat Assessment Framework

### 🔴 High Threat Criteria
- Direct competition in same segment (grades 3-8 assessment)
- Strong brand + good execution + similar price point
- Recent funding or growth momentum
- Could easily pivot to compete directly

### 🟡 Moderate Threat Criteria
- Some overlap but different primary focus
- Strong in one area (brand OR execution OR pricing)
- Could potentially compete but would require significant changes
- Financial resources to enter our space

### 🟢 Low Threat Criteria
- Different business model or segment
- Execution problems despite good concept
- Unlikely to pivot to our space
- Not competing for same customers

## Example Workflow

- Web search for company info, products, pricing

- Organize information into section structure
- Identify 5-7 strengths with evidence
- Identify 7-10 weaknesses with user complaints

### Step 3: Strategy (30-45 minutes per competitor)

## Common Pitfalls to Avoid

❌ **Don't**: Just describe what competitors do
✅ **Do**: Analyze why it matters and how AceQuest can differentiate

❌ **Don't**: Copy marketing speak from competitor websites
✅ **Do**: Find real user reviews and quote actual feedback

❌ **Don't**: Assess all competitors as "high threat"
✅ **Do**: Realistically evaluate based on actual overlap and execution

❌ **Don't**: Write generic "opportunities" like "be better"
✅ **Do**: Specify exactly what AceQuest will do differently

❌ **Don't**: Ignore competitor strengths or only focus on weaknesses
✅ **Do**: Objectively assess both to understand true competitive position

❌ **Don't**: Use outdated information or guess at details
✅ **Do**: Research current information and cite sources

❌ **Don't**: Write one long essay without structure
✅ **Do**: Use clear sections, tables, and bullet points for readability
